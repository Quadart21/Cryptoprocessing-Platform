# v0.9.0 — 2026-05-08

Тестовые webhook по конкретному инвойсу для мерчантов, ответ сервера в модалке, и пагинация длинных списков на обзоре платформы.

## Добавлено

### Backend

- **`POST /client/invoices/{invoice_id}/webhook-test`** (право `client.webhooks.write`): отправка на настроенный URL проекта JSON в формате, близком к боевому уведомлению о платеже.
  - Событие **`invoice.test_deposit`**, в корне **`simulated: true`** и поясняющий **`message`** — чтобы отличить от продакшена.
  - В теле передаются актуальные **`invoice`** и при наличии последняя **`transaction`** по этому инвойсу.
  - **Статус инвойса в БД не меняется**, записи о доставке webhook в системе не создаются.
  - Ответ API **всегда 200** при обработке запроса: в теле — **`status_code`**, **`response_preview`** (до 500 символов), **`ok`** (успешный HTTP 2xx у мерчанта), **`error`** при сетевых ошибках / после ретраев, плюс **`event_id`**, **`attempts`**, **`webhook_url`**.

### Frontend (мерчант)

- В разделе **Инвойсы** у каждой строки кнопка **Webhook** (активна при настроенном webhook URL и праве на webhook).
- Модалка **«Уведомление о пополнении (симуляция)»**: отправка теста и показ **ответа сервера мерчанта** (HTTP, превью тела, ошибка транспорта при необходимости).

### Frontend (платформа)

- **Последние инвойсы платформы** и **Последние операции** на обзоре: по **10** элементов на страницу, счётчик «Показано X–Y из Z», кнопки навигации.

## Изменено

- Оформление блока пагинации и модалки теста webhook (`merchant-workspace.css`, `dashboard.css`).
